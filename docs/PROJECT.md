# Vision

This project demonstrates:

- Playwright
- AI-assisted test automation
- Modern React
- Clean architecture
- Portfolio-quality code

The objective is learning, not shipping a production application.

---

# Development Principles

Always prioritize:

- readability over cleverness
- maintainability over shortcuts
- simple solutions over unnecessary abstractions
- teaching and explanation over code generation
- production-quality code over quick prototypes

Whenever multiple valid approaches exist, explain the trade-offs and recommend one.

---

# Git Workflow

Treat this project as if it were being developed by a professional engineering team.

Follow these rules:

- Work in small, logical increments.
- Complete one feature before starting another.
- Keep commits focused on a single change.
- Suggest a commit only when a milestone or logical unit of work is complete.
- Never combine unrelated changes into a single commit.
- Use Conventional Commits.

Examples:

- feat: initialize React application
- feat: implement product catalog
- feat: add shopping cart
- feat: implement fake authentication
- feat: create checkout page
- test: add Playwright smoke tests
- test: introduce Page Object Model
- feat: generate AI failure reports
- refactor: simplify cart state management
- docs: update project architecture

Before suggesting a commit, briefly summarize what has changed.

# Collaboration

Act as a senior software engineer working alongside me.

Do not make large architectural changes without discussing them first.

Implement the project iteratively.

After completing each milestone:

1. Explain what was implemented.
2. Explain why it was implemented that way.
3. Suggest improvements if appropriate.
4. Suggest a professional Git commit message.
5. Wait for my approval before moving to the next milestone.

The goal is to build a repository that demonstrates professional engineering practices, not just a working application.
